## Docker 란 

구글에 검색하면 **"애플리케이션을 신속하게 구축, 테스트, 배포할 수 있는 오픈 소스 컨테이너화 플랫폼"**이라는 사전적 정의가 나온다. <br/> 
하지만 직접 써보기 전까지는 이미지, 컨테이너 같은 용어가 전혀 와닿지 않는다.<br/> 
<br/> 
쉽게 말해, 도커는 배포 환경의 **'불확실성'**을 제거하기 위해 등장했다.
<br/> 

---

### 🛠️ 왜 도커를 쓰는가? (OS와 설치의 늪)

프론트엔드든 백엔드든 개발을 하면 결국 배포를 해야 한다. 그런데 이 배포라는 게 생각보다 까다롭다.

윈도우냐 리눅스냐에 따라 설치 방법이 제각각이다.

자바 버전 하나만 달라도 실행이 안 되는 경우가 부지기수다.

서버 환경이 바뀔 때마다 WAS, 웹 서버 설정을 새로 하느라 진을 다 뺀다.

도커는 이 복잡한 과정을 **"그냥 OS 위에 도커 하나 깔고 끝내자"**로 해결했다. 도커만 있으면 어떤 OS에서든 명령어로 필요한 환경을 즉시 구축하고 공유할 수 있기 때문이다.

---

### 🎮 메이플스토리로 이해하는 도커 개념

글로만 보면 어려우니, 게임에 비유해 보자.

메이플 스토리는 몬스타를 잡아서 레벨업하는 RPG 게임이고, 몬스터를 잡을때마다 정해진 경험치를 획득한다. <br/>
하지만 사설(프리)서버의 메이플스토리를 보면 경험치 100배인곳도 존재하고, 드랍율이 100배인 곳도 존재한다. <br/>
<br/>

메이플스토리 페이지에 접속하여 다운로드 하면 넥슨 런처(=도커 엔진)가 다운로드 된다. <br/>
그 후 넥슨 런처를 실행하면 메이플 스토리(=도커 이미지) 설치가 진행 된다. <br/>
설치 완료 후 시작 버튼을 누르면 게임 서버(=도커 컨테이너)에 접속 된다. <br/>


1. 도커 엔진 = 넥슨 런처 <br/>
게임을 실행하고 관리하는 기본 프로그램. 이게 깔려 있어야 게임을 할 수 있다.

2. 도커 이미지 = 설정이 완료된 게임 스냅샷 <br/>
운영체제, 라이브러리, 설정이 모두 포함된 '불변의 폴더'. <br/>
이미 패치가 다 되어 있어서 별도의 설치 과정 없이 즉시 실행 가능하다.

3. 도커 컨테이너 = 실제로 실행된 게임 월드 <br/>
이미지를 실행한 상태. 월드 안에서 무슨 짓을 하든(파일을 지우든 몬스터를 잡든) 원본 이미지(설계도)는 변하지 않는다. 마음에 안 들면 월드를 끄고 다시 새 월드를 열면 초기 상태로 시작된다.

4. 도커 허브 = 전 세계 유저들의 커스텀 설정 공유소
"누가 경험치 100배 설정 다 끝내놨네? 나도 이거 받아서 바로 써야지!" 하고 가져오는 곳.
