## Docker란 무엇인가?

사전적으로는 **"애플리케이션 컨테이너화 플랫폼"**이라고 하지만, <br/> 
처음 접하는 사람에게는 와닿지 않는 말이다. <br/>
도커는 단순히 프로그램 하나가 아니라, 여러 기술 요소가 모여 배포 환경의 **'불확실성'**을 제거하는 하나의 생태계를 의미한다.<br/>
<br/>
💡 도커라는 포괄적인 개념을 한 번에 삼키려 하기보다, 각 구성 요소를 먼저 이해한 뒤 "아, 이것들이 모여서 도커가 되는구나!"라고 깨닫자.<br/>
<br/>
본인은 **'도커'** 에 포커스를 맞추게 이해하려다보니 중간에 여러번 포기할뻔했다. <br/>

## Docker 의 구셩요소

> 이해하기 쉽도록 대중적인 게임에 비유를 하자.


| 구성 요소 | 기술적 정의 (Technical) | 게임 비유 (MapleStory) |
| :--- | :--- | :--- |
| **Docker Engine** | 도커의 심장. 컨테이너를 생성하고 실행하는 **실제 구동부** | **넥슨 런처 / 게임 실행기** |
| **Docker Image** | 앱 실행에 필요한 모든 것이 압축된 **읽기 전용 상태 (스냅샷)** | **설정 완료된 게임 폴더** (배율 100배 등) |
| **Docker Container** | 이미지를 실행하여 생성된 **격리된 가상 공간 (프로세스)** | **현재 접속 중인 월드** (실제 게임 플레이 중) |
| **Docker Hub** | 이미지를 저장하고 전 세계 유저와 공유하는 **원격 저장소** | **프리서버 설정 공유 게시판** |
| **Docker Compose** | 여러 컨테이너를 정의하고 한꺼번에 실행하는 **관리 도구** | **공대원 구성** |

### 🛠️ 왜 도커를 쓰는가? (OS와 설치의 늪)

프론트엔드든 백엔드든 개발을 하면 결국 배포를 해야 한다. 그런데 이 배포라는 게 생각보다 까다롭다.

윈도우냐 리눅스냐에 따라 설치 방법이 제각각이다.

자바의 설치 방법도 다를뿐더러, 버전 하나만 달라도 실행이 안 되는 경우가 부지기수다.

서버 환경이 바뀔 때마다 WAS, 웹 서버 설정을 새로 하느라 진을 다 뺀다.

도커는 이 복잡한 과정을 **"그냥 OS 위에 도커 하나 깔고 끝내자"**로 해결했다. <br/>
도커만 있으면 어떤 OS에서든 명령어로 필요한 환경을 즉시 구축하고 공유할 수 있기 때문이다.

### 🎮 메이플스토리로 이해하는 도커 개념

글로만 보면 어려우니, 게임에 비유해 보자.

메이플 스토리는 몬스타를 잡아서 레벨업하는 RPG 게임이고, <br/>
몬스터를 잡을때마다 정해진 경험치를 획득한다. <br/>
<br/>
하지만 사설(프리)서버의 메이플스토리를 보면 경험치 100배인곳도, 드랍율이 100배인 곳도 존재한다. <br/>
<br/>
메이플스토리 페이지에 접속하여 다운로드 하면 넥슨 런처(=도커 엔진)가 다운로드 된다. <br/>
그 후 넥슨 런처를 실행하면 메이플 스토리(=도커 이미지) 설치가 진행 된다. <br/>
설치 완료 후 시작 버튼을 누르면 게임 서버(=도커 컨테이너)에 접속 된다. <br/>


1. 도커 엔진 = 넥슨 런처 <br/>
게임을 실행하고 관리하는 기본 프로그램. 이게 깔려 있어야 게임을 할 수 있다.

2. 도커 이미지 = 설정이 완료된 게임 스냅샷 <br/>
운영체제, 라이브러리, 설정이 모두 포함된 '불변의 폴더'. <br/>
이미 패치가 다 되어 있어서 별도의 설치 과정 없이 즉시 실행 가능하다.

3. 도커 컨테이너 = 실제로 실행된 게임 월드 <br/>
이미지를 실행한 상태. 월드 안에서 무슨 짓을 하든(파일을 지우든 몬스터를 잡든) 원본 이미지(설계도)는 변하지 않는다. 마음에 안 들면 월드를 끄고 다시 새 월드를 열면 초기 상태로 시작된다.

4. 도커 허브 = 전 세계 유저들의 커스텀 설정 공유소
"누가 경험치 100배 설정 다 끝내놨네? 나도 이거 받아서 바로 써야지!" 하고 가져오는 곳.
